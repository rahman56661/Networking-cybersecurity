ğŸ“¦

Efficient IP Usage

Without subnetting, a company needing 300 hosts would need to waste an entire Class B block (65,534 IPs)! Subnetting lets us carve networks into exactly the right sizes. Security Isolation

Place HR, Finance, and Engineering on separate subnets. A firewall or router between subnets controls what traffic flows between them. A virus in the Engineering subnet can't automatically spread to Finance if they're on different subnets with ACLs. VLANs + subnets = defense in depth for enterprise security.

Broadcast Control

Broadcasts in TCP/IP go to ALL devices on the same subnet. If you have 1,000 devices on one giant subnet, every broadcast reaches all 1,000 â€” massive overhead! Splitting into 10 subnets of 100 devices limits each broadcast to only 100 devices. Smaller subnets = less broadcast noise = better performance.

Hierarchical Routing

Subnets allow routers to aggregate routes. Instead of knowing about every individual IP, a router just needs to know "all 192.168.x.x addresses are reachable via this interface." This route summarization massively shrinks routing tables and speeds up routing decisions. The internet's routing tables are manageable only because of subnet aggregation.

A MAC Address (Media Access Control Address) is a unique hardware identifier assigned to every network interface card (NIC) at the time of manufacture. Unlike IP addresses (which are logical and changeable), MAC addresses are burned into the hardware â€” they are physical, permanent identifiers.

While IP addresses work at Layer 3 (Network layer) and handle routing between networks, MAC addresses work at Layer 2 (Data Link layer) and handle delivery within a single network segment. When your PC sends data to another PC on the same LAN, the switch uses MAC addresses â€” not IP addresses â€” to deliver the frame to the right port.

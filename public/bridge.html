üîç What is a Network Bridge?

A network bridge is an intelligent Layer 2 (Data Link layer) device that connects two or more LAN segments and filters traffic based on MAC addresses. Unlike a repeater which just amplifies signals, or a hub which broadcasts to all ports, a bridge learns which devices are on which segment and only forwards traffic when necessary.

üîß Why Bridges Are Needed

Bridges solve several important networking problems:

  • Collision Reduction: Splits large collision domain into smaller ones - fewer collisions per segment
  • Performance Improvement: Each segment only sees its own traffic plus cross-segment traffic
  • Network Segmentation: Logically separates network while maintaining connectivity
  • Distance Extension: Can extend network beyond single segment limits
  • Bandwidth Multiplication: Each segment gets full bandwidth, not shared
üè¢ Real-World Example - Office Network:

Problem: 50 computers in office on single Ethernet hub = constant collisions, very slow.

Solution with Bridge:
1. Split into 2 segments: Engineering (25 PCs) and Marketing (25 PCs)
2. Install bridge between segments
3. Bridge learns: Engineering PCs on Segment A, Marketing PCs on Segment B
4. When Engineering PC1 talks to Engineering PC2 ‚Üí stays on Segment A (bridge blocks)
5. When Engineering PC1 sends file to Marketing PC1 ‚Üí bridge forwards to Segment B
6. Each segment now has only 25 devices ‚Üí 50% less collisions on each side
7. Network performance doubles! üöÄ

Result: Same 50 PCs, but 2 separate collision domains = much faster network.

üéØ Core Functions:

  • MAC Address Learning: Observes source MAC addresses and learns which segment each device is on
  • Frame Filtering: Only forwards frames when destination is on different segment
  • Frame Forwarding: Passes frames between segments when needed
  • Loop Prevention: Uses Spanning Tree Protocol to prevent network loops
  • Collision Domain Separation: Each port/segment is separate collision domain

‚öôÔ∏è How Bridge Works - Detailed Operation

Bridges operate at Layer 2 and make intelligent forwarding decisions based on MAC addresses:

1Ô∏è‚É£

Learning Phase

Bridge starts with empty MAC address table. Observes all frames passing through. Examines source MAC address of each frame. Records "MAC address X is on Port/Segment Y" in its table. Builds complete map of network over time. Entries age out if not seen for period (default 300 seconds).

2Ô∏è‚É£

Filtering Decision

When frame arrives, bridge checks destination MAC. Looks up destination in MAC table. If source and destination on SAME segment ‚Üí FILTER (don't forward). If on DIFFERENT segments ‚Üí FORWARD (send across). If destination unknown ‚Üí FLOOD (send to all segments except source).

3Ô∏è‚É£

Forwarding Frame

Frame needs to cross segments. Bridge buffers entire frame in memory. Waits for destination segment to be clear (no collision). Retransmits frame on destination segment. Source segment and destination segment operate independently - separate collision domains.

4Ô∏è‚É£

Error Checking

Bridge examines Frame Check Sequence (FCS). Drops corrupted frames - doesn't forward bad data. Filters out collision fragments. May discard malformed frames. Only forwards valid, clean frames. Improves overall network quality.

üì¶ Example Frame Processing:

Scenario 1 - Frame Filtered (Same Segment):
1. PC A1 (AA:01 on Segment A) sends to PC A2 (AA:02 on Segment A)
2. Bridge receives frame on Segment A port
3. Checks source: AA:01 ‚Üí updates "AA:01 on Segment A" (already known)
4. Checks destination: AA:02 ‚Üí looks in table ‚Üí "AA:02 on Segment A"
5. Decision: Source and destination both on Segment A ‚Üí FILTER ‚ùå
6. Frame NOT forwarded to Segment B ‚Üí reduces traffic on Segment B ‚úÖ

Scenario 2 - Frame Forwarded (Different Segments):
1. PC A1 (AA:01 on Segment A) sends file to PC B1 (BB:01 on Segment B)
2. Bridge receives frame on Segment A port
3. Checks destination: BB:01 ‚Üí looks in table ‚Üí "BB:01 on Segment B"
4. Decision: Destination on different segment ‚Üí FORWARD ‚úÖ
5. Bridge buffers frame, waits for Segment B clear, retransmits
6. PC B1 receives file successfully! üéâ

üì¶ Types of Bridges

üîó

Transparent Bridge

Most common type. Completely invisible to connected devices - they don't know bridge exists. Automatically learns topology. No configuration of end devices needed. Plug and play operation. Uses Spanning Tree Protocol. Standard in Ethernet networks.

üõ£Ô∏è

Source Routing Bridge

Used in Token Ring networks (obsolete). Source device determines entire path. Frame contains routing information. Bridge just follows instructions in frame. More complex, rarely used today. Replaced by transparent bridging in modern networks.

üì°

Wireless Bridge

Connects wired LAN to wireless LAN or two wireless LANs. Operates as Layer 2 bridge using Wi-Fi. Useful for connecting buildings without running cables. Can bridge Ethernet to Wi-Fi transparently. Common in campus environments and industrial settings.

üåê

Remote Bridge

Connects LANs across WAN link (e.g., two offices in different cities). Uses point-to-point connection like leased line or VPN. Makes remote LANs appear as single network. Each location has local bridge. Useful for extending corporate networks geographically.

üîÑ

Translation Bridge

Connects different network types (e.g., Ethernet to Token Ring). Translates between different frame formats. Complex due to differences in protocols. Mostly obsolete - modern networks standardized on Ethernet. Historical importance but rare today.

üñ•Ô∏è

Software Bridge

Implemented in software on computer with multiple network interfaces. Turn PC/server into bridge. Linux, Windows, BSD support bridging. Useful for virtualization and testing. Flexible but slower than hardware bridges. Good for lab environments.

‚úÖ Advantages of Bridges

üéØ

Collision Domain Segmentation

Each port/segment is separate collision domain. Drastically reduces collisions. Better performance for all devices. If 50 devices, splitting into 2 segments = ~75% fewer collisions per segment. Network becomes much faster and more reliable.

üöÄ

Performance Improvement

Filters unnecessary traffic between segments. Each segment only sees local + cross-segment traffic. Reduces bandwidth waste. Better utilization of available capacity. Can double or triple effective network speed. Users experience faster file transfers and less lag.

üß†

Intelligent Forwarding

Makes smart decisions based on MAC addresses. Only forwards when necessary. Learns network topology automatically. No manual configuration of MAC tables. Adapts to network changes. Much smarter than hub or repeater - actual intelligence in operation.

üëª

Transparent Operation

Devices don't know bridge exists. No configuration changes needed on end devices. Works with any protocol. Plug and play from device perspective. Easy deployment in existing networks. Backward compatible with all Ethernet devices.

üîí

Improved Security

Traffic isolation between segments. Devices on Segment A can't see Segment B's local traffic. Reduces packet sniffing opportunities. Can help contain security incidents to one segment. Better than hub where everyone sees everything.

üìè

Distance Extension

Can extend network beyond single segment limits. Each segment can be full length (100m for Ethernet). Bridge regenerates signal like repeater. Total distance: segment 1 + segment 2 + ... Useful for large facilities.

üõ°Ô∏è

Error Filtering

Drops corrupted frames - doesn't forward bad data. Filters collision fragments. Improves overall network quality. Bad frames stay on local segment. Prevents error propagation. Healthier network overall.

üîÑ

Backward Compatibility

Works with old and new devices. No upgrades needed on end stations. Can connect legacy equipment to modern network. Protects existing investment. Gradual network evolution possible. Very flexible.

‚ùå Disadvantages & Limitations

üí∞

Higher Cost Than Hub/Repeater

More expensive than simple repeaters or hubs. Intelligence requires more complex hardware. CPU, memory for MAC table needed. Still cheaper than router but pricier than Layer 1 devices. However, performance improvement usually justifies cost.

üåä

Broadcast Domain Still Shared

All segments share single broadcast domain. Broadcast traffic forwarded to all segments. ARP requests flood entire bridged network. Large bridged networks still suffer broadcast storms. Need router to segment broadcast domains. Can't scale infinitely.

‚è±Ô∏è

Adds Latency

Store-and-forward operation adds delay. Must receive entire frame before forwarding. Processing MAC table takes time. Latency higher than hub (no processing). Modern switches faster but bridge still adds microseconds to milliseconds delay.

üîÑ

Cannot Route Between Networks

Layer 2 only - works with MAC addresses. Cannot route between different IP networks. Cannot perform NAT. Cannot connect to Internet directly. Need router for inter-network communication. Limited to extending single network, not connecting multiple networks.

üîÅ

Spanning Tree Overhead

STP prevents loops but blocks redundant links. Convergence time after topology change. BPDU packets consume bandwidth. Complex in large networks. Misconfiguration can cause network outages. Need to understand STP to manage properly.

üìä

Limited Scalability

MAC tables have size limits. Large networks exhaust table capacity. Performance degrades with too many segments. Spanning Tree becomes complex. Better to use routers and Layer 3 switches for large networks. Bridges work best in small to medium LANs.

üö´

No Protocol Filtering

Forwards all Layer 2 protocols. Cannot filter by IP address or port. No firewall capability. Cannot prioritize traffic (no QoS). All filtering is MAC-based only. Need Layer 3+ devices for advanced filtering.

üìà

Learning Delay

MAC table starts empty. Initial traffic may be flooded. Takes time to learn all MAC addresses. Topology changes require relearning. During learning phase, more traffic than necessary. Eventually stabilizes but initial period less efficient.

üí° Key Insight:

A switch is essentially a multi-port bridge with hardware acceleration. They work on the same principles (MAC learning, filtering, forwarding) but switches are optimized for connecting many devices with high performance. The term "bridge" is mostly historical - today we just use switches for the same purpose.

üéØ Key Takeaways

  • A bridge is a Layer 2 device that connects LAN segments and filters traffic based on MAC addresses
  • Bridges learn which devices are on which segment by observing source MAC addresses
  • Smart filtering: only forwards frames when source and destination on different segments
  • Creates separate collision domains - dramatically reduces collisions and improves performance
  • Operates transparently - devices don't know bridge exists, no configuration needed
  • Store-and-forward: receives entire frame, checks for errors, then forwards if needed
  • Much more intelligent than hub or repeater, but simpler than router
  • Modern switches are multi-port bridges with hardware acceleration
  • All segments share same broadcast domain - broadcasts forwarded to all segments
  • Used Spanning Tree Protocol (STP) to prevent network loops
